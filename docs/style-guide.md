# Sunless Sea 繁中翻譯風格指南

> 適用於在對話中直接翻譯（每批 60–80 條），以及人工審閱。

---

## 一、語氣定調

Sunless Sea 的文字是**維多利亞哥德 × 克蘇魯宇宙恐怖 × 黑色幽默**的混合體，不是純粹的其中一種。

**三層文字感：**

| 層次 | 特質 | 中文對應策略 |
|------|------|------|
| 維多利亞哥德 | 正式、有距離感、偽學術語氣 | 書面語，帶古典色彩詞彙 |
| 克蘇魯恐怖 | 暗示大於描寫，語言本身製造陌生感 | 幽冥、蒼茫、晦暗等有「積塵感」的詞 |
| 黑色幽默 | 冷靜口吻說可怕的事，反差製造效果 | 輕描淡寫，絕不翻死，保留反差 |

**核心原則：**
- 精煉優先——每個字框都很短，要在有限字數內保留氣氛，不冗贅
- 語氣要冷靜，帶距離感——主體是宇宙，不是人
- 暗示比描述有力——不把恐怖說完，讓讀者去填空

---

## 二、詞彙：「積塵感」策略

英文克蘇魯原文刻意選用古舊、冷僻詞彙（如 `eldritch`、`cyclopean`），讓語言本身帶有異質感。
中文讀者對「不可名狀」已無感，**要用帶有古典漢語色彩的詞彙重建陌生感**。

**建議詞彙庫（視情況選用）：**

| 概念 | 優先選用 | 避免 |
|------|----------|------|
| 黑暗 | 幽冥、晦暗、玄冥、冥暗 | 很黑、非常暗 |
| 廣大/深邃 | 蒼茫、浩渺、無垠、深淵 | 很大、很深 |
| 怪異 | 詭譎、悖理、莫名、悚然 | 奇怪、很怪 |
| 古老 | 亙古、太古、不知年歲 | 很老、古老的 |
| 無法理解 | 非人所能言說、其形難辨、難以名狀 | 無法形容、說不出來 |
| 恐懼 | 悚然、凜懼、毛骨悚然 | 很害怕 |
| 海洋相關 | 浪湧、深海、幽淵、暗海 | 很深的海 |

**文言點綴原則：** 不要全文言，只在關鍵意象、強調句、怪物描述時自然帶入。用錯文言比不用更糟。

---

## 三、「不可名狀」的哲學立場

洛夫克拉夫特的「不可名狀」（unnameable / unspeakable）**不是懶惰的退路，是哲學立場**：
人類語言本來就不足以描述宇宙真相，語言的邊界即是人類認知的邊界。

翻譯時，遇到刻意模糊的描述，**不要補充說明，不要把意思說清楚**。保留留白。

```
EN: something vast and nameless stirred in the deep
TC（正確）: 某個龐大而無名的存在在幽淵中蠕動
TC（錯誤）: 一個巨大的怪物在深海裡動了起來   ← 把怪物說死了
```

---

## 四、黑色幽默的處理

Sunless Sea 常用**冷靜、平淡的口吻描述可怕的事**，這種反差是核心效果，翻譯時必須保留。

```
EN: Lost With All Hands — You, and all your crew, are gone.
    London will speculate in vain as to how you might have met your end.

TC（正確）: 全員陣亡——你和所有船員，就此消失。
           淪敦的人們將徒勞地猜測，你究竟以何種方式走向終局。

TC（錯誤）: 你和所有船員都死了！倫敦的人不知道你們是怎麼死的。
```

幽默不是要你加笑點，而是**輕描淡寫**，讓讀者自己感受到諷刺。

## 五、人稱代詞

| 英文 | 繁中 | 說明 |
|------|------|------|
| you（玩家/船長） | 你 | 不用「您」。冷靜的第二人稱，帶命運感 |
| your | 你的 | |
| Captain（NPC 稱呼玩家）| 船長 | |
| I / me（NPC 對話）| 我 | |
| he / she（性別不明 NPC）| 他們 | The Alarming Scholar 刻意性別模糊，統一用「他們」 |

## 六、標點符號

### 基本規則
- 全形標點：，。！？；：（）「」『』
- 引號：外層「」，內層『』，不用 "" 或 ''
- 破折號：——（兩個全形），對應英文 em dash 或連續 dash
- 刪節號：……（六個點），對應英文 ellipsis（...）

### 特殊情境
- 英文以 `-` 製造停頓或未完成的句子 → 譯為——
- 句末未完成時（原文以 ` - ` 結尾）→ 以——結尾，製造氣息感
- 全大寫強調（`THE SUN!`）→ 用感嘆號處理，不用全大寫

### 對話引號
所有對話一律改用「」：
- `'Sorry, captain.'` → 「對不起，船長。」
- `"I need to talk to Penstock"` → 「我要去找潘斯托克。」

---

## 七、格式標籤（必須保留）

```
<i>斜體</i>     → 保留標籤，內容翻譯
<br/>           → 完整保留，不翻譯
[遊戲機制提示]   → 方括號保留，內容翻譯
{x} {y}         → 佔位符完整保留，不翻譯
```

機制提示範例：
- EN: `[This card will continue to appear until you bury your father's bones]`
- TC: `[此卡將持續出現，直到你安葬父親的骸骨]`

### ChangeDescriptionText / LevelDescriptionText 格式

這兩個欄位使用特殊分隔格式，**必須原樣保留分隔符，只翻譯文字部分**：

```
格式：level|文字~level|文字~level|文字

EN: "1|Welcome aboard, stranger.~0|A certain matter has become clear."
TC: "1|歡迎登船，陌生人。~0|某件事已然清晰。"
```

- `level|` 前的數字是觸發等級，**不翻譯，不修改**
- `~` 是條目分隔符，**完整保留**
- 如果欄位為空字串或 null，保持原樣不動

## 八、地名與人名策略

### 核心術語（固定，見 glossary/terms.json）

| 英文 | 繁中 | 備注 |
|------|------|------|
| Fallen London | 淪敦 | 陷落後的倫敦，淪（沉淪）的諧音 |
| London（陷落前）| 倫敦 | 僅用於歷史指稱，如「三十年前的倫敦」 |
| the Unterzee / zee | 海澤 | 德語 Unter + 荷蘭語 Zee |
| the Neath | 尼斯 | 音譯，beneath 縮略，傳遞異質感 |
| the Bazaar | 集市 | |
| the Masters | 主人們 | |
| Captain | 船長 | |

### 淪敦 vs 倫敦：語境判斷規則

- **現在的城市**（遊戲中的主要港口）→ **淪敦**
- **陷落前的歷史指稱**（如「三十年前」、「第六城」、Surface 上的城市）→ **倫敦**
- 不確定時看語境：若談的是「現在這座沉入地下的城市」，用淪敦

### 地名翻譯策略

**所有地名一律翻譯為中文，不得保留英文原文。**

- 有描述性意義：意譯為主（如 Avid Horizon → 渴望的地平線）
- 語源為希臘／拉丁等古語：依語源意譯（如 POLYTHREME → 萬嚎之境）
- 梵語、特殊語源音譯保留異質感（如 Varchas → 縛遮娑，Irem → 伊雷姆）
- 無明確語義：音意混譯，選擇有異質感的漢字

已確定地名完整清單見 `glossary/terms.json`。

### NPC 名字
- 職稱型名字：翻譯職稱（The Alarming Scholar → 驚人的學者）
- 象徵型名字：意譯（The Sorrow-Spider → 悲哀之蛛）
- 純人名：音譯（Penstock → 潘斯托克）

**同樣不得保留英文原名，一律給予中文譯名。**

### 品名 / 卡片名的四字詞原則

物品名、卡片名適合壓成四字，增加節奏感與質感，**但不強制**：
- `A Memory of Distant Shores` → 遠岸追憶
- `A Zee-Ztory` → 海澤軼事
- `Vision of the Surface` → 地表風景

若四字後意義有損失，優先保留意義。

## 九、海澤六色（Neathbow）

這六種顏色是 Fallen London 世界觀的核心術語，**不可當作普通顏色詞翻譯**，固定使用以下譯名：

| 英文 | 繁中 | 詞源 |
|------|------|------|
| Apocyan | 起源藍 | apo（起源/離開）+ cyan（藍綠），記憶與珊瑚之藍 |
| Peligin | 黯澤黑 | pelagus（海）+ fuligin（比黑更深之色），最深之海 |
| Violant | 銘刻紺 | violet/violent，血在尖頂之地流淌，紺色似紅非紅 |
| Irrigo | 沒齒紫 | 拉丁 irrigo（淹沒/擴散），染遍被遺忘角落 |
| Gant | 枯寂褐 | guant，所有顏色被吞噬後的遺骸 |
| Cosmogone | 宇宙金 | cosmogony（天體演化）+ gone（消逝），記憶中太陽的顏色 |
| Viric | 淺眠綠 | 鏡子背後的顏色，淺眠之色 |

遇到這些詞時，直接替換，無需另行翻譯。

## 十、遊戲術語翻譯規則

### quality / qualities

「quality」是遊戲內部用語，涵蓋物品、能力、故事旗標等，**絕對不翻成「品質」**。

| quality 的實際身份 | 翻譯策略 |
|------|------|
| 物品、貨物、寶物 | 直接給中文名稱，不加分類標籤 |
| 角色能力（Iron / Mirrors 等）| 能力、屬性 |
| 故事進度旗標 | 進度、記錄 |
| 文字中直接出現 "quality" | 改為「記錄」或「標記」 |

```
EN: Your "A Stranger" quality has gone.
TC（正確）: 你的「陌生人」記錄已消失。
TC（錯誤）: 你的「陌生人」品質已消失。
```

## 十一、禁翻欄位（技術性字串）

以下**絕對不翻譯**：
- `glossary/protected.json` 中列出的 ID 的 `Name` 欄位
- 槽位名稱：Forward / Deck / Aft / Engines / Bridge / Auxiliary
- 程式識別字：`ImageName`、`Tag`、`PrefabName` 等欄位
- 佔位符：`{x}`、`{y}`、`_REUSE`

## 十一、翻譯工作流備忘

**當前工作流（不使用 API）：**
```
source/en/flat/{file}_flat.json
    ↓ 在對話中直接翻譯（每批 60–80 條）
source/en/flat/{file}_translated.json
    ↓ python3 scripts/merge_translations.py {type}
translations/vanilla/{file}.json
    ↓ 複製到 addon/vanilla/
遊戲測試
```

**翻譯時帶入的上下文：**
1. 本風格指南第一～四節
2. `glossary/terms.json` 術語表（特別是海澤六色、地名）
3. `glossary/protected.json` 禁翻清單

**每次只改以下欄位的值：**
`Name`、`Description`、`Teaser`、`ButtonText`、`ExoticEffects`、
`ChangeDescriptionText`、`LevelDescriptionText`、`EnhancementsDescription`、
`SaleDescription`、`BuyMessage`、`SellMessage`、`LevelImageText`

其餘欄位結構完整保留，不得增減、不得修改鍵名。
